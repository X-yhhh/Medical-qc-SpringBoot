# 使用说明

更新时间：2026-03-13

## 1. 适用对象

本文档面向系统使用者，按医生和管理员两类角色说明当前系统的主要操作路径。

## 2. 登录与退出

### 2.1 登录

1. 访问系统登录页
2. 输入用户名、密码和身份
3. 登录成功后进入主界面

### 2.2 注册

1. 在登录页进入注册页
2. 填写用户名、邮箱、密码、姓名、医院、科室和角色
3. 提交注册信息后返回登录

### 2.3 退出或切换用户

1. 点击右上角用户下拉菜单
2. 选择“切换用户”或“退出登录”

## 3. 医生端使用流程

### 3.1 查看仪表盘

登录后可在仪表盘查看：

- 今日待处理任务数
- 质控趋势
- 风险预警
- 最近访问
- 任务快捷入口

### 3.2 执行头部出血检测

1. 进入“头部出血检测”
2. 选择“本地影像上传”或 “PACS 系统调取”
3. 填写患者姓名、患者编号、检查编号、性别、年龄、检查日期
4. 本地模式下上传图片文件，当前支持 `PNG/JPG/JPEG/BMP`
5. 提交后等待 AI 分析完成
6. 查看风险评分、主要异常和检测详情
7. 如有需要可重新分析或导出报告

注意：

- 当前模型暂不支持直接上传 DICOM 序列
- PACS 模式依赖缓存中的检查记录

### 3.3 提交其他质控任务

1. 进入对应任务页面，如头部平扫、胸部平扫、胸部增强或冠脉 CTA
2. 上传文件或填写任务信息
3. 对于 `CT头部平扫质控`、`CT胸部平扫质控`，需补齐患者姓名、检查号、性别、年龄、检查日期
4. 提交任务后到“质控任务中心”查看状态和结果

说明：

- 当前 `CT头部平扫质控`、`CT胸部平扫质控` 已为真实模型推理链路
- 当前 `CT胸部增强质控`、`冠脉CTA质控` 仍为 mock 链路
- 所有任务都统一进入任务中心、统一结果表和异常工单链路

### 3.4 查看任务中心

在“质控任务中心”可：

- 按任务类型筛选
- 按状态筛选
- 查看任务来源
- 查看任务详情与结果摘要

### 3.5 管理患者信息

1. 进入对应的患者信息管理页
2. 按患者编号、姓名、检查号查询
3. 新增或编辑患者信息
4. 必要时上传影像附件
5. 可执行“从 PACS 同步”初始化当前任务类型的患者数据

### 3.6 处理异常工单

1. 进入“异常汇总”
2. 查看总异常数、趋势和异常分布
3. 在列表中搜索患者姓名或检查号
4. 打开工单详情
5. 更新状态、处理人、根因、纠正措施、预防措施和备注
6. 保存后完成流转

## 4. 管理员端使用流程

### 4.1 查看全局仪表盘和任务中心

管理员可查看：

- 全局任务状态
- 风险预警
- 异常工单
- 系统运行趋势

### 4.2 用户管理

1. 进入“用户管理”
2. 按关键字、角色、启用状态查询用户
3. 编辑用户角色、状态和资料

### 4.3 规则中心

1. 进入“规则中心”
2. 按模块或关键字查询现有规则
3. 新增规则或编辑规则
4. 设置异常项、优先级、责任角色、SLA
5. 设置是否自动建单和启用状态

## 5. 常见使用建议

- 出血检测前优先确认患者编号和检查编号，避免历史数据混淆
- 工单处理时尽量补全根因、纠正措施和预防措施，便于后续复盘
- 规则中心修改后，应观察新任务是否按预期生成异常工单
- 若 PACS 查询无数据，先确认缓存表中是否已有对应检查记录

## 6. 常见问题

### 6.1 登录成功后页面跳回登录页

可能原因：

- Redis 未启动
- 会话失效
- 账号角色与目标菜单权限不匹配

### 6.2 头部出血检测无法完成

可能原因：

- Python 推理服务未启动
- 上传文件格式不支持
- 本地文件或网络连接异常

### 6.3 PACS 搜索没有结果

可能原因：

- 对应任务专属 PACS 缓存表中没有对应记录
- 输入的患者编号、姓名或检查号不匹配

### 6.4 异常工单没有自动生成

可能原因：

- 当前任务未命中规则
- 规则未启用
- 自动建单开关关闭
